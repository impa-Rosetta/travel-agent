# AI Travel Agent 开发规则

## 项目愿景

AI Travel Agent 是一个长期学习型 AI Agent 项目。

项目目标不是快速生成代码，而是在真实项目中逐步理解 AI Agent 架构、软件工程和产品开发。

这个项目应该一步一步成长。每一次开发任务，都应该同时推进产品能力和开发者理解。

## 开发理念

1. 不一次性开发完整项目。
2. 每次只完成一个小模块。
3. 开发前先解释设计思路。
4. 代码优先简单清晰。
5. 不为了炫技引入复杂框架。
6. 不修改无关文件。
7. 每个功能必须可以独立提交 Git。

## 每日开发流程

1. 明确今日目标。
2. 解释涉及知识。
3. 设计实现方案。
4. 编写代码。
5. 测试。
6. 更新开发日志。
7. 提供 Git Commit 信息。

## 学习记录要求

每次开发完成后，需要记录：

- 使用了什么技术
- 为什么使用
- 核心概念是什么
- 数据如何流动
- 后续如何扩展
